# SKILLS & PROCESS: TODO App Project

このプロジェクトを通じて培われたスキル、およびAI（Antigravity）との効果的な共同開発プロセスについてまとめます。

## 🧠 習得したスキル

### 1. AIへの高精度な指示（プロンプト設計）
- **具体性の向上**: 「きれいに」だけでなく「VisionOS風」「フロストガラス」「抜け感」「0.5pxのボーダー」といった具体的なデザイン言語を使うことで、期待以上の結果を引き出すスキル。
- **反復的な改善（Iterative Refinement）**: 一度の指示で完成を目指すのではなく、「スクロールを減らして」「マウスエフェクトを消して」といった粒度の細かいフィードバックによる調整力。

### 2. モダンUIデザインの実装知識
- **Glassmorphism**: 透過、ぼかし（`backdrop-filter`）、および彩度（`saturate`）を組み合わせた高級感のあるUIの実装。
- **タイポグラフィの深掘り**: フォントウェイトを細くし、レタースペーシングを広げることで「空気感」を演出する手法。

### 3. デバッグと構造理解
- **フルスタックの全体像**: フロントエンドのデザイン変更が、どのようにサーバーを経由してローカルの `todo.md` に書き込まれるかというシステムの流れの理解。

### 4. クラウドデプロイとレスポンシブ
- **Vercelによる世界公開**: CLIを用いたサーバーレスデプロイと、Web上での本番環境の構築。
- **モバイルファーストの視点**: デスクトップだけでなく、スマホユーザーを意識した「ポチポチしやすい（Tap Target）」設計とメディアクエリの実装。

## 🔧 工夫したプロセス

### プロトタイピングの段階的アプローチ
- 最初から100点のデザインを目指さず、まずは「機能」を固め、その後に「第一弾デザイン（宇宙）」、さらに「洗練デザイン（自然）」、そして「最終章（スマホ最適化・デプロイ）」へと段階的にアップグレードした。これにより、機能と美しさと利便性を確実なものにした。

### コンテンツとデータの分離
- タスク内容（`todo.md`）とデザイン（CSS）を分けることで、データの管理はMarkdownで手軽に行い、表示は最先端のWeb技術で彩るという、開発者とユーザーの利便性を両立させた。
